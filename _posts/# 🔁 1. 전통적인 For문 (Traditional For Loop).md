# # 🔁 1. 전통적인 For문 (Traditional For Loop)
전통적인 for문은 **반복 횟수가 명확**하거나 **배열/리스트의 인덱스에 직접 접근**해야 할 때 사용합니다.
### 📌 특징 및 구조
* **구조:** 초기화, 조건식, 증감식 세 부분이 명확하게 분리되어 있습니다.
* **활용:** 배열이나 컬렉션의 특정 인덱스에 접근하여 값을 **수정**할 수 있습니다.
* **반복 횟수:** 반복 횟수를 자유롭게 조절할 수 있습니다 (예: 2씩 증가시키기, 거꾸로 반복하기).

⠀💻 문법
```Java
for (초기화; 조건식; 증감식) {
    // 조건식이 참일 때 실행할 코드
}
```

### 💡 예시 (배열 인덱스를 사용한 반복)
```java
Java

String[] names = {"김", "이", "박"};

for (int i = 0; i < names.length; i++) {
    // i는 인덱스 (0, 1, 2)
    System.out.println(i + "번째 이름: " + names[i]);
    
    // 인덱스를 사용하여 배열의 요소 수정 가능
    if (i == 1) {
        names[i] = "최";
    }
}

```


# ⚡ 2. 향상된 For문 (Enhanced For Loop / For-each Loop)
향상된 for문은 자바 5부터 도입되었으며, **배열이나 컬렉션의 모든 요소를 순서대로 접근**할 때 코드를 **간결하게** 작성할 수 있도록 해줍니다.
### 📌 특징 및 구조
* **구조:** 반복 대상 (배열 또는 컬렉션)과 요소를 저장할 변수만 지정합니다.
* **활용:** 배열이나 컬렉션의 요소를 **읽기(순회)**만 할 때 사용합니다. **인덱스에 직접 접근할 수 없으므로, 요소의 값을 수정할 수는 없습니다.**
* **가독성:** 코드가 매우 간결해져 가독성이 높습니다.

⠀💻 문법
Java
```java
for (요소타입 변수명 : 배열 또는 컬렉션) {
    // 배열/컬렉션에서 하나씩 꺼낸 요소(변수명)를 사용하여 실행할 코드
}
```

### 💡 예시 (요소를 순회하는 반복)
Java

String[] names = {"김", "이", "박"};
```java
for (String name : names) {
    // name은 배열에서 하나씩 꺼낸 요소 ("김", "이", "박")
    System.out.println("이름: " + name);
}
```


# 🆚 3. 두 반복문의 주요 차이점
| **구분** | **전통적인 For문** | **향상된 For문** |
|---|---|---|
| **인덱스 접근** | **가능** (인덱스 i 사용) | **불가능** (요소만 접근) |
| **요소 수정** | **가능** (인덱스를 통해) | **불가능** (순회용으로만 사용) |
| **반복 제어** | 반복 횟수/방향을 **자유롭게 제어** 가능 | 항상 **처음부터 끝까지** 순서대로 순회 |
| **사용 용도** | 인덱스가 필요하거나 반복 횟수 제어가 필요할 때 | 배열/컬렉션의 모든 요소를 **순회(읽기)**만 할 때 |
| **가독성** | 복잡도가 높음 | **매우 높음** (간결함) |
### 🎯 선택 기준
* **인덱스가 필요 없는 단순 순회:** **향상된 for문**을 사용하세요. (가장 좋은 선택)
* **인덱스를 사용하여 요소를 수정하거나 특정 위치에 접근해야 할 때:** **전통적인 for문**을 사용하세요.
