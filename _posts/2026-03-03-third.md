### 변수의 종류 및 특징 비교



---

변수의 종류에 따른 특징을 비교한 표입니다.

| **구분** | **인스턴스 변수** | **클래스 변수 (Static)** | **지역 변수 (Local)** |
|:---:|:---:|:---:|:---:|
| **선언 위치** | 클래스 내부 (메서드 밖) | 클래스 내부 (static) | 메서드/생성자 블록 내부 |
| **키워드** | 없음 | **static** | 없음 |
| **생명 주기** | **객체** 생성 ~ 소멸 | **클래스** 로드 ~ 종료 | 블록 실행 시작 ~ 종료 |
| **저장 위치** | 힙(Heap) 영역 | 메서드 영역 (Static) | 스택(Stack) 영역 |
| **공유 여부** | 객체마다 **고유** | 모든 객체가 **값 공유** | 해당 블록 내에서만 유효 |
| **초기화** | **자동** 초기화 | **자동** 초기화 | **수동**으로 초기화 필수 |

표에 대한 설명이 끝났습니다.

# 변수 종류별 상세 특징


### 1\. 인스턴스 변수 (Instance Variables)

* **용도:** 객체(Object)마다 **고유하게** 가져야 하는 속성(상태)을 정의할 때 사용됩니다.
* **예시:** Car 클래스에서 각 차마다 다른 **색상(color)**이나 **차량 번호(carNumber)**를 저장하는 변수.
* **메모리:** 객체가 저장되는 **힙(Heap) 영역**에 위치합니다.

⠀
### 2\. 클래스 변수 (Class Variables / Static Variables)

* **용도:** 그 클래스에 속하는 **모든 객체가 공통적으로 공유**해야 하는 속성이나 값을 정의할 때 사용됩니다.
* **예시:** Car 클래스에서 모든 차가 동일한 **바퀴 개수(wheelCount = 4)**를 가지거나, 객체가 몇 개 생성되었는지 카운트하는 변수.
* **메모리:** 클래스 정보가 저장되는 **메서드 영역(Method Area)**에 위치합니다. 따라서 객체 생성 없이도 접근 가능합니다.

⠀
### 3\. 지역 변수 (Local Variables)

* **용도:** 메서드나 블록 내부에서 **일시적으로** 사용하고 버릴 필요가 있는 데이터를 저장할 때 사용됩니다.
* **예시:** calculate() 메서드 내에서 임시 계산 결과를 저장하는 tempResult 변수.
* **메모리:** 메서드가 호출될 때 생성되는 **스택(Stack) 영역**에 위치하며, 메서드 실행이 끝나면 자동으로 제거됩니다.
* **특이점:** 컴파일러는 지역 변수의 자동 초기화를 지원하지 않으므로, 개발자가 값을 할당하지 않고 사용하려 하면 **컴파일 오류**가 발생합니다.
